Bug Report Example (From Real Experience)

Bug Report: Bill Payment Success Message Displays Before Backend Processing Completes

Bug ID: BUG-PAY-2019-047 (Actual bug I found at IBERIABANK)

Severity: Critical | Priority: P1

Module: Online Banking - Bill Payment

Environment: Production, Chrome 76, Windows 10

Description:

The online bill payment system displays "Payment Successful" confirmation message immediately after user clicks submit, but the backend processing occasionally fails without updating the frontend. This results in customers believing their bills are paid when transactions never actually processed.

Steps to Reproduce:

  1. Log into online banking as test user (account #987654321)
  2. Navigate to "Pay Bills"
  3. Select any registered payee
  4. Enter payment amount: $150.00
  5. Select payment date: Today
  6. Click "Submit Payment"
  7. Observe "Payment Successful" message
  8. Check transaction history - payment may not appear
  9. Check backend database - no payment record exists

Expected Result:

System should wait for backend confirmation before displaying success message. If backend processing fails, user should see error message: "Payment could not be processed. Please try again."

Actual Result:

Frontend displays "Payment Successful" based on form submission alone, not backend confirmation. Approximately 3-5% of payments fail silently.

SQL Evidence:

IT team confirmed via database query: No payment record existed despite "success" message shown to customer

Customer Impact:

  • Customers receive late fees from billers
  • 10+ customer complaints per week
  • Reputational damage to bank
  • Potential regulatory compliance issues

Root Cause Analysis:

Frontend JavaScript triggers success message on form submit event rather than waiting for API response from backend payment processing service.

Recommended Fix:

Implement async/await pattern to wait for backend API confirmation before displaying success message. Add proper error handling and timeout logic (30 seconds).

// Current (broken)
submitPayment() { showSuccessMessage(); }

// Should be
async submitPayment() {
try {
const result = await processPayment();
if (result.success) { showSuccessMessage(); }
} catch (error) { showErrorMessage(); }
}
